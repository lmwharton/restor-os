05b · Editing an existing pin (material / dry_standard / location)

Rare, but always possible

Dry standard and material are set at placement but stay editable — the tech might realize mid-job that a wall is moisture-resistant drywall (needs dry_standard bumped from 16 to 21), or that they picked the wrong material entirely. This flow is secondary — hidden behind one tap, not daily chrome.

Daily

Reading value. Tap pin → reading sheet → type number → save. Every day.

Rare

Material + dry standard. Set at placement, edited only if categorization was wrong.

Never

Position on canvas. The physical spot doesn't move. (If wrong, delete + re-drop.)

Answer to "how does editing work?"
Where to edit From the reading sheet → tap Edit pin in the metadata row. Secondary UI — never competes with the daily Save reading action.
What's editable Surface, Position, Material, Dry standard. (Location's x/y is not — to move a pin, delete and re-drop.)
When to use it Rarely. Only when the initial material categorization was wrong (moisture-resistant drywall, cement backer, custom override).
Confirm modal Fires only when dry_standard changed. Shows which colors will flip. Otherwise (e.g., renaming location only) saves silently.
Backend endpoint PATCH /v1/jobs/{jobId}/moisture-pins/{pinId} — already built, already wired in the useUpdateMoisturePin hook.
05 · Implementation checklist

Every web/CLAUDE.md rule, mapped to an element

If any of these deviate during coding, we stop and revisit. No quiet drift.

The 8 placement decisions
✓Mode toggle in Row 1 (between Save and pills)
✓Tool palette swaps content per mode
✓Canvas unchanged — rooms/walls dim to 0.30
✓Pins on top Konva layer
✓Placement → bottom sheet (mobile) / centered modal (desktop)
✓Reading → same pattern as placement
✓ConfirmModal inside the reading sheet (overlay, not separate fullscreen)
✓Tap-outside-room → inline nudge, 1.6s fade
web/CLAUDE.md conventions honored
✓Selection chips rounded-lg, h-7 sm:h-9
✓Action buttons h-10 rounded-lg flex-1 inside sheet/modal
✓Back arrow w-10 h-10 rounded-xl bg-surface-container-low
✓Cards bg-surface-container-lowest rounded-xl
✓Material dropdown uses select pattern (h-9 rounded-lg)
✓Sub-page headers text-lg font-semibold
✓Sticky footer border-t, outside scroll
✓No native confirm → <ConfirmModal>
✓Same chip shape at every breakpoint (no rounded-full on mobile)
✓Geist Sans + Geist Mono typography
Platform-specific patterns
Mobile Sheet slides from bottom · handle · sticky action footer · full-width (flex-1) Cancel + Save
Desktop Centered modal over canvas backdrop · width 440px · sticky footer with compact buttons (inline-flex px-5, h-9)
